Test Scenarios

Scenario 1: Coffee Shop with Classic Theme

  1. Open the Website Builder
  2. Select "Coffee Shop" from business type dropdown
  3. Click "Use Preset" button
  4. Verify tagline: "Your Daily Dose of Happiness"
  5. Verify description mentions artisan coffee
  6. Verify menu items: Espresso (Rp 15,000), Cappuccino (Rp 25,000), Latte (Rp 28,000), Croissant (Rp 18,000)
  7. Fill in business name, WhatsApp, and Instagram
  8. Select "Classic" theme
  9. Click "Generate Website"
Expected Result: 4 files download (index.html, styles.css, menu.json, deployment-instructions.txt). Generated site has light, professional styling with coffee shop content.

Scenario 2: Food Stall with Kiroween Theme

  1. Refresh the page or clear the form

  6. Generate website
Expected Result: Website generates successfully with manually entered data. Presets are optional.

Scenario 9: Deployment Instructions Validation

  1. Generate any website
  2. Locate and open deployment-instructions.txt
  3. Verify it contains clear file explanations
  4. Verify Netlify drag-and-drop instructions are present
  5. Verify GitHub Pages instructions are present
  6. Verify other hosting options mentioned (Vercel, Cloudflare Pages)
  7. Verify update instructions are included
  8. Verify troubleshooting tips are included
  9. Check that language is simple and non-technical
Expected Result: Deployment instructions are clear, comprehensive, and easy to follow for non-technical users.

Scenario 10: Responsive Design - Mobile

  1. Generate any website
  2. Open generated index.html
  3. Open browser DevTools (F12)
  4. Toggle device toolbar (Ctrl+Shift+M or Cmd+Shift+M)
  5. Set viewport to 320px width (iPhone SE)
  6. Verify layout adapts to mobile
  7. Verify text is readable
  8. Verify buttons are at least 44px
  9. Verify menu items stack vertically
Expected Result: Website displays perfectly on mobile with no horizontal scrolling, readable text, and touch-friendly buttons.

Scenario 11: Responsive Design - Tablet & Desktop

  1. With generated site open in DevTools
  2. Set viewport to 768px (iPad)
  3. Verify layout adapts appropriately
  4. Set viewport to 1024px (Desktop)
  5. Verify layout uses full width
  6. Set viewport to 1920px (Large Desktop)
  7. Verify content is centered and readable
Expected Result: Website adapts smoothly to all viewport sizes with appropriate layouts for each breakpoint.
